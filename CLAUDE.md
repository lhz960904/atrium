# Atrium · Project Instructions

1. **Always follow community best practice** — research first, cite the evidence (issue / doc / thread), decide second.
2. **Always pin to the latest stable major version** of every package; bump majors when they ship. On conflict with Rule 1, use the version Rule 1 prescribes.
3. **Comment placement decides the style.** Inside a function body, prefer single-line comments. At a function head/top, or for hack / trick / non-obvious logic, use a multi-line comment that fully explains the why — multi-line comments aid both AI and human comprehension, so don't truncate them where understanding is at stake.
4. **Any Vercel AI SDK code MUST be preceded by reading the official AI SDK docs** (ai-sdk.dev) and using the latest documented best practice — before writing or changing it
